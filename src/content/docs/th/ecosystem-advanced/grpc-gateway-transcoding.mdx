## สองกลุ่มผู้ใช้ contract เดียว

ในโมดูล foundations เราพูดไว้ว่าระบบเดียวกันมักอยากได้ gRPC ภายในและ REST ที่ขอบนอก — call ที่เร็วและ typed ระหว่าง service ของตัวเอง แต่เป็น JSON ธรรมดาสำหรับ third party, browser และการเช็คด้วย `curl` เร็ว ๆ

**Transcoding** ให้คุณ serve ทั้งสองจาก `.proto` *เดียวกัน* คุณ annotate แต่ละ method ด้วย REST route ที่มันควรตอบด้วย แล้ว **reverse proxy** ที่ generate มาจะแปล request แบบ JSON/HTTP ที่เข้ามาให้เป็น gRPC call และแปลง protobuf response กลับเป็น JSON

## Annotate ที่ method

คุณอธิบาย REST mapping ไว้ใน service definition เลย โดยใช้ `google.api.http` annotation:

```proto
import "google/api/annotations.proto";

service UserService {
  rpc GetUser(GetUserRequest) returns (User) {
    option (google.api.http) = {
      get: "/v1/users/{id}"
    };
  }

  rpc CreateUser(CreateUserRequest) returns (User) {
    option (google.api.http) = {
      post: "/v1/users"
      body: "*"
    };
  }
}
```

ตอนนี้ `GetUser` ตอบ `GET /v1/users/42` ได้ด้วย — path segment `{id}` map เข้ากับ field `id` ของ request ส่วน `CreateUser` ตอบ `POST /v1/users` โดย JSON body map เข้ากับ request message ตัว gRPC method ไม่เปลี่ยน annotation แค่เพิ่มประตู REST เข้าไป

gateway เป็น code ที่ generate มา (grpc-gateway ใน ecosystem ของ Go; Envoy ก็ทำ transcoding ผ่าน filter ได้) มัน parse URL กับ JSON, สร้าง protobuf request, เรียก gRPC method จริง แล้ว marshal reply กลับเป็น JSON — ทั้งหมดขับเคลื่อนด้วย annotation

## เมื่อไรควรหยิบมาใช้

- **มี consumer ภายนอกหรือ browser** ที่คาดหวัง REST/JSON แต่คุณไม่อยาก maintain API ตัวที่สองด้วยมือ transcoding ทำให้เหลือ contract เดียว
- **อยากได้ความ `curl`-ได้ และ OpenAPI doc** annotation ชุดเดียวกัน generate OpenAPI spec ได้
- **ข้ามไปถ้าทุกอย่างอยู่ภายใน** ถ้ามีแต่ service ของตัวเองที่เรียก API การเพิ่ม proxy hop และการแปลง JSON เป็น overhead ล้วน ๆ — อยู่กับ native gRPC ต่อไป

<Callout title="สรุปหนึ่งบรรทัด">
Transcoding serve ทั้ง REST/JSON และ gRPC จาก `.proto` เดียว: `google.api.http` annotation ประกาศ REST route ให้แต่ละ method และ reverse proxy ที่ generate มา (grpc-gateway หรือ Envoy) แปล JSON↔protobuf ใช้เมื่อต้องหันหน้าออกโลกภายนอกโดยไม่ต้อง maintain API ตัวที่สอง — และข้ามไปเมื่อทุก caller เป็น internal gRPC อยู่แล้ว
</Callout>
