## enum: ชุด value ที่มีชื่อ

**enum** คือ field ที่เป็นได้เพียงหนึ่งในชุด integer ที่มีชื่อกำหนดไว้ตายตัว มันสื่อ intent และกัน state ที่ไม่ถูกต้องได้ดีกว่า `int32` หรือ `string` เปล่า ๆ มาก

```proto
enum Status {
  STATUS_UNSPECIFIED = 0;
  STATUS_ACTIVE      = 1;
  STATUS_SUSPENDED   = 2;
  STATUS_CLOSED      = 3;
}

message Account {
  int64  id     = 1;
  Status status = 2;
}
```

## กฎ zero value

มีกฎหนึ่งข้อที่ข้ามไม่ได้: **value แรกของ enum ต้องเป็น `0` และควรหมายถึง "unspecified"** เพราะ proto3 ไม่มี null ให้ scalar field enum ที่ไม่ถูก set จะอ่านได้เป็น zero value ถ้า `0` แปลว่า `ACTIVE` ทุก account ที่ลืม set status จะดูเหมือน active แบบเงียบ ๆ

## nested message

message ใส่ message อื่นข้างในได้ การ nest จัดกลุ่มข้อมูลที่เกี่ยวข้อง และให้ reuse type หรือจำกัด scope ของ type ไว้กับ message ที่เป็นเจ้าของ

```proto
message Order {
  int64 id = 1;

  message LineItem {          // scope อยู่ใน Order (Order.LineItem)
    string sku      = 1;
    int32  quantity = 2;
  }

  repeated LineItem items = 2;
  Address shipping        = 3;  // message ระดับบนสุดที่ reuse มาใช้ที่นี่
}
```

nest type เมื่อมันมีความหมายเฉพาะตอนอยู่ใน parent (`Order.LineItem`) และเก็บไว้ระดับบนสุดเมื่อหลาย message ใช้ร่วมกัน (`Address`)

## oneof: เลือกได้อย่างเดียวจากหลาย field

**oneof** บอกว่า "set ได้มากสุดหนึ่ง field ในกลุ่มนี้ต่อครั้ง" การ set ตัวหนึ่งจะล้างตัวอื่น และ generated code ให้วิธีเช็กชัด ๆ ว่าตัวไหนถูก set

```proto
message Notification {
  int64 user_id = 1;

  oneof channel {
    string email        = 2;
    string sms_number   = 3;
    string push_token   = 4;
  }
}
```

ตรงนี้ notification ส่งออกผ่าน channel เดียวเป๊ะ ๆ `oneof` model "tagged union" ได้แม่นยำ — ดีกว่า field nullable สามตัวกับ comment ที่หวังว่าจะมีตัวเดียวถูกกรอกเยอะ และยังประหยัด memory เพราะ field เหล่านั้นใช้ storage ร่วมกัน

<Callout title="สรุปหนึ่งบรรทัด">
ใช้ **enum** กับชุด value ที่ตายตัว — แต่ให้ value `0` หมายถึง `UNSPECIFIED` เสมอ ไม่งั้น field ที่ไม่ถูก set จะปลอมเป็นค่าจริง **nest** message ที่เป็นของ parent เดียว และเก็บตัวที่ใช้ร่วมกันไว้ระดับบนสุด ใช้ **oneof** เมื่อควร set ได้ตัวเดียวจากหลาย field เพื่อให้ schema บังคับกฎ "เลือกหนึ่ง" แทนที่จะเป็น comment ที่ได้แต่หวัง
</Callout>
