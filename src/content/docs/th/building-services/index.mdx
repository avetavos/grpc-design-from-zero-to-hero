## จาก contract ไปสู่ code ที่รันได้

คุณมี `.proto` contract แล้ว โมดูลนี้จะเปลี่ยนมันให้เป็นของที่รันได้จริง: server ที่ตอบ call, client ที่ยิง call และวินัยเรื่อง deadline ที่กันไม่ให้ทั้งระบบค้างไปตลอดกาล

ทุกบทเรียนใช้ contract เล็ก ๆ ตัวเดียวกัน เพื่อให้คุณตามตัวอย่างเดียวได้ตั้งแต่ต้นจนจบ:

```proto
syntax = "proto3";
package user.v1;

message GetUserRequest {
  int64 id = 1;
}

message User {
  int64 id = 1;
  string name = 2;
  string email = 3;
}

service UserService {
  rpc GetUser(GetUserRequest) returns (User);
}
```

## โมดูลนี้ครอบคลุมอะไรบ้าง

| บทเรียน | สิ่งที่คุณจะได้เรียนรู้ |
|--------|-------------------|
| Code Generation | `protoc`, plugin ของแต่ละภาษา, `buf` และสิ่งที่ generate ออกมาจริง ๆ |
| Implementing a Server | เติม handler, register service และ serve |
| Writing a Client | channel, stub, การยิง call และการ reuse connection |
| Deadlines & Cancellation | ทำไมทุก call ต้องมี deadline และมันส่งต่อกันอย่างไร |

สังเกตว่า generated code อยู่ตรงกลาง: มันยื่น interface ให้ **server** ไป implement และยื่น stub ให้ **client** ไปเรียก คุณเขียนสองฝั่ง ส่วน codegen รับประกันว่าทั้งสองฝั่งตรงกัน

<Callout title="The one-line takeaway">
การสร้าง gRPC service มี 3 จังหวะ: generate typed code จาก `.proto`, implement server handler และเรียกมันจาก client stub — แล้วห่อทุก call ด้วย deadline เพื่อไม่ให้อะไรค้างไปตลอดกาล
</Callout>
