## คอร์สนี้ทำงานยังไง

<CardGrid>
  <Card title="Contract-first เสมอ" icon="seti:json">
    ทุกบทเรียนเริ่มจาก `.proto` — contract ที่เป็นกลางไม่ผูกกับภาษา — แล้วค่อยแสดง server และ client ที่ generate ออกมา เพื่อให้เห็นว่า schema ขับเคลื่อน code อย่างไร
  </Card>
  <Card title="สามภาษาเทียบกันชัด ๆ" icon="seti:config">
    ตัวอย่าง server และ client ใน Go, Python และ Node (TypeScript) ผ่านแท็บที่ sync กัน เลือกภาษาครั้งเดียวแล้วตามไปทุกที่
  </Card>
  <Card title="เน้นการออกแบบ ไม่ใช่แค่ syntax" icon="approve-check">
    ทุกหัวข้อครอบคลุมเหตุผลและ trade-off — เมื่อไรควร stream, ตั้ง deadline อย่างไร, evolve schema อย่างไรโดยไม่ทำให้ client พัง
  </Card>
  <Card title="ไดอะแกรมที่อธิบายได้" icon="puzzle">

  </Card>
